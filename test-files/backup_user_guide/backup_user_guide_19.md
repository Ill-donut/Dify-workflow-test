# Deleting restore points

Restore points can be deleted automatically or manually depending on how they were generated.

- Restore points generated by automatic execution are kept according to the retention policy. The restore points exceeding the retention range will be deleted automatically. Alternatively, you can delete them manually.
- Restore points generated by manual execution are kept permanently by default. You can only delete them manually.

> **Note**:
>
> When you manually delete restore points, consider the following:
>
> - If the restore point is generated by a full backup, all incremental restore points that depend on this restore point will be deleted together with the selected restore point.
> - If the restore point is generated by an incremental backup, the full and incremental restore points it depends on and all incremental restore points that depend on it will be deleted along with the selected restore point.

**Prerequisite**

When deleting restore points manually, ensure that there are no ongoing backup or restore jobs related to these restore points.

**Procedure**

1. In the left sidebar of the **Backup & DR** page in CloudTower, click **Backup plan**. Select a backup plan and click its name to enter its details page.

2. Navigate to the **VM restore point** tab and click the virtual machine name. On the right side of the page, delete its restore points in the following ways.

   - Click **...** to the right of a restore point. Select **Delete** to delete that restore point.

   - Check the checkbox to the left of a restore point, click **Delete** in the upper-right corner to delete one or multiple restore points.

   - Click **...** on the right side of the page. Select **Delete all restore points** to delete all restore points for the virtual machine.

3. In the pop-up dialog box, read and confirm the prompt message. Then, click **Delete** to complete the deletion.
